What makes it different?

Why OCaml?

* One core lib, used in both the web server and cli

* Explicit handling of error cases

* Ecosystem is very helpful.

* Building on windows is surprisingly easy.

* Most of the important code is a functor over a provider signature.

Infra in OCaml

* More people should write infra code in OCaml.

* Infra is hard, strong types make it easier.

* Handling failure for infra is very hard.

Warning: BETA

* We use this in production, that doesn't mean you should.

* Still pre 1.0 release.

* Still bugs and bad error messages all over the place.
